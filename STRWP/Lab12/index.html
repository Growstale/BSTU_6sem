API Сервер (18-01)

Этот сервер предоставляет RESTful API для управления данными об университете (факультеты, кафедры, аудитории и т.д.), хранящимися в базе данных VAV на сервере VODCHYTS.

Используйте инструменты типа Postman для взаимодействия с API.

Доступные эндпоинты:

Факультеты (Faculties)

  • GET /api/faculties - Получить все факультеты
  • POST /api/faculties - Добавить новый факультет (JSON в теле: {"FACULTY": "код", "FACULTY_NAME": "имя"})
  • PUT /api/faculties - Обновить факультет (JSON в теле: {"FACULTY": "код", "FACULTY_NAME": "новое_имя"})
  • DELETE /api/faculties/:facultyCode - Удалить факультет по коду

Кафедры (Pulpits)

  • GET /api/pulpits - Получить все кафедры
  • POST /api/pulpits - Добавить новую кафедру (JSON: {"PULPIT": "код", "PULPIT_NAME": "имя", "FACULTY": "код_факультета"})
  • PUT /api/pulpits - Обновить кафедру (JSON: {"PULPIT": "код", "PULPIT_NAME": "новое_имя", "FACULTY": "новый_код_факультета"})
  • DELETE /api/pulpits/:pulpitCode - Удалить кафедру по коду

Дисциплины (Subjects)

  • GET /api/subjects - Получить все дисциплины
  • POST /api/subjects - Добавить новую дисциплину (JSON: {"SUBJECT": "код", "SUBJECT_NAME": "имя", "PULPIT": "код_кафедры"})
  • PUT /api/subjects - Обновить дисциплину (JSON: {"SUBJECT": "код", "SUBJECT_NAME": "новое_имя", "PULPIT": "новый_код_кафедры"})
  • DELETE /api/subjects/:subjectCode - Удалить дисциплину по коду

Типы Аудиторий (Auditorium Types)

  • GET /api/auditoriumstypes - Получить все типы аудиторий
  • POST /api/auditoriumstypes - Добавить новый тип (JSON: {"AUDITORIUM_TYPE": "код", "AUDITORIUM_TYPENAME": "имя"})
  • PUT /api/auditoriumstypes - Обновить тип (JSON: {"AUDITORIUM_TYPE": "код", "AUDITORIUM_TYPENAME": "новое_имя"})
  • DELETE /api/auditoriumtypes/:typeCode - Удалить тип по коду

Аудитории (Auditoriums)

  • GET /api/auditoriums - Получить все аудитории
  • POST /api/auditoriums - Добавить новую аудиторию (JSON: {"AUDITORIUM": "код", "AUDITORIUM_NAME": "имя", "AUDITORIUM_CAPACITY": вместимость, "AUDITORIUM_TYPE": "код_типа"})
  • PUT /api/auditoriums - Обновить аудиторию (JSON: {"AUDITORIUM": "код", "AUDITORIUM_NAME": "новое_имя", "AUDITORIUM_CAPACITY": новая_вместимость, "AUDITORIUM_TYPE": "новый_код_типа"}) (Исправлено с /auditorims)
  • DELETE /api/auditoriums/:auditoriumCode - Удалить аудиторию по коду (Исправлено с /auditorims)

Примечание: Примеры JSON в теле запроса могут потребовать указания всех обязательных полей согласно схеме БД.
